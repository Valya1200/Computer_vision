Алгоритм выполнения морфологической эрозии изображения
1. Подготовка данных

Первый этап - загрузка исходного цветного изображения в формате BGR и преобразование его в градации серого. Для этого используется взвешенное суммирование каналов по стандарту ITU-R BT.601: зеленый канал получает вес 0.587, красный 0.299, синий 0.114. Полученное градационное изображение содержит значения яркости от 0 до 255 для каждого пикселя.

2. Пороговая бинаризация
На втором этапе градационное изображение преобразуется в бинарное с помощью пороговой бинаризации при пороге 128. Все пиксели с яркостью 128 и выше становятся белыми (255), пиксели с яркостью меньше 128 становятся черными (0). Это создает четкое разделение объекта и фона, необходимое для морфологических операций.

3. Создание структурирующего элемента
Структурирующий элемент представляет собой квадратную матрицу 5x5, полностью заполненную единицами. Этот элемент определяет размер окрестности, в которой будет выполняться морфологическая операция. Размер 5x5 означает, что анализ проводится по 25 соседним пикселям вокруг каждого центрального пикселя.

4. Выполнение эрозии вручную
Эрозия реализуется следующим образом. Для каждого пикселя результирующего изображения извлекается окно размером 5x5 из дополненного исходного изображения. Пиксель в результирующем изображении принимает значение 255 только в том случае, если абсолютно все пиксели в этом окне равны 255. Если хотя бы один пиксель в окне имеет значение 0, то центральный пиксель результата становится 0.
Перед сверткой изображение дополняется нулевой рамкой шириной 2 пикселя с каждой стороны, что соответствует радиусу структурирующего элемента.

5. Результат обработки
В результате применения эрозии белые объекты на бинарном изображении уменьшаются в размере. Границы объектов отступают внутрь примерно на 2 пикселя с каждой стороны. Тонкие элементы могут полностью исчезнуть, а шум в виде мелких белых точек гарантированно удаляется.

## 3.2 выравнивание гистограммы

Выравнивание гистограммы предназначено для повышения контрастности изображения путем перераспределения яркостных значений таким образом, чтобы гистограмма интенсивностей была равномерно растянута по всему доступному диапазону (0-255). Это позволяет максимально использовать динамический диапазон изображения и улучшить различимость деталей.

Входные данные
Градационное изображение размером MxN пикселей с уровнями яркости от 0 до 255
Общее количество пикселей N = M × N
Алгоритм 
Этап 1. Построение гистограммы частот
Пример: h(100) = 1500 пикселей имеют яркость 100
Этап 2. Вычисление кумулятивной функции распределения (CDF)

![Функция распределения](https://github.com/Valya1200/Computer_vision/blob/main/fig8.png?raw=true)

Рисунок 8 – Функция распределения

Этап 3. Нормализация CDF
Полученная кумулятивная функция нормализуется к диапазону 0-255 по формуле:
new_intensity(k) = CDF(k) × 255 / total_pixels 
new_intensity(k) — новое значение яркости для всех пикселей, имевших исходную яркость k.
Этап 4. Замена яркостей пикселей
Каждый пиксель изображения с исходной яркостью r заменяется на новое значение согласно построенному отображению:
Если пиксель имел яркость r, то получает яркость 
s_r = CDF(r) × 255 / total_pixels
Результат обработки
После выполнения алгоритма гистограмма результирующего изображения становится равномерно распределенной по всему диапазону 0-255. Кумулятивная функция распределения (CDF) выровненного изображения представляет собой прямую линию, что соответствует идеальному равномерному распределению яркостей.

## 3.3 поворот изображений на угол кратный 90 градусов
Цель операции
Операция поворота предназначена для изменения ориентации изображения на заданный угол (90°, 180°, 270°) относительно исходного положения. Поворот выполняется без интерполяции с сохранением точных значений пикселей.
Для выполнения операций поворота изображения на углы 90°, 180° и 270° применялась встроенная функция библиотеки NumPy np.rot90().
Описание функции np.rot90(img, k, axes=(0,1))
Параметры:
img — входное изображение 
k — количество поворотов на 90° по часовой стрелке:
k=1 — поворот на 90° против часовой стрелке
k=2 — поворот на 180°
k=3 — поворот на 90° по часовой (= 90° против часовой)
axes=(0,1) — поворачиваются первые две размерности (высота, ширина)
