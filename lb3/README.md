# Модель усредненного фона

Поскольку мышка движеться в постоянном фоне, наилучшим решением данной задачи будет являться метод построения модели фона. 
Была создана функция sr_mod_fon(), которая вычисляет усредненный фон путем суммирования значений пикселей по всем кадрам и деления на количество кадров. Полученная модель фона отображает статическую часть сцены.

![two_mice](https://github.com/Valya1200/Computer_vision/blob/main/images/two_mice.png?raw=true)

По итогу, нам удалось получить карту фона, полученную усреднением. На карте видно контур мышки в 2-х местах. Это происходит потому, что мышка в этих местазх достаточно долго сидит и не двигается. Это затрудняет определение. Поэтому вычитание пикселей между картой фона и изображением (для примера возьмем 2000-чное) мы будем использовать пороговую бинаризацию по разнице.

# Вычитание текущего кадра от фона

Для выделения движущегося объекта (мыши) из текущего кадра (кадр №2005) была вычтена модель фона. Полученная разностная изображение была преобразована в бинарное с помощью пороговой обработки (порог 20). Для улучшения качества бинарного изображения были применены морфологические операции: Открытие (удаление шума), закрытие (заполнение пробелов в объекте)

 # Модель фона и отслеживание движение мыши, кадр за кадром
 
Была создана функция mod_fon(), которая в цикле обрабатывает все кадры видео:
Используется экспоненциальное скользящее среднее для адаптивного обновления модели фона (50% текущего кадра + 50% предыдущей модели)
Для каждого кадра вычисляется разность с фоном, применяется пороговая обработка
Находятся моменты и координаты центра масс
Добавлены проверки для коррекции ошибок: Если объект не найден (координаты (0,0)), используются координаты из предыдущего кадра. Если объект "прыгает" более чем на 30 пикселей (слишком быстрое движение), используются предыдущие координаты
