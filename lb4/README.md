# Лабораторная работа №4 

Цель: Разработать классификатор

## ШАГ 1 Загрузка данных 
За основу был выбран датасет "Обнаружение дорожных знаков" на kaggle. Датасет состоит из 15 классов: Зеленый свет, красный свет, ограничение скорости 10, ограничение скорости 100, ограничение скорости 110, ограничение скорости 120, ограничение скорости 20, ограничение скорости 30, ограничение скорости 40, ограничение скорости 50, ограничение скорости 60, ограничение скорости 70, ограничение скорости 80, ограничение скорости 90, стоп

## Шаг 2: Настройка параметров аугментации
INFER_HEIGHT = 32   # Высота входного изображения
INFER_WIDTH = 32    # Ширина входного изображения

## Шаг 3: Определение функций аугментации

Cоздаются функции для аугментации (искусственного увеличения обучающей выборки) тренировочных и валидационных данных. Аугментация включает:
HorizontalFlip - зеркальное отражение по горизонтали
VerticalFlip - зеркальное отражение по вертикали
ShiftScaleRotate - сдвиг, масштабирование и поворот
RandomBrightnessContrast - изменение яркости и контраста
HueSaturationValue - изменение цветовой гаммы
GaussNoise - добавление случайного шума
CoarseDropout - случайное вырезание прямоугольных областей

## Шаг 4: Создание генератора данных
Создается класс-генератор, который:

Загружает данные порциями (батчами) для экономии памяти

Применяет аугментацию "на лету" во время обучения

Перемешивает данные после каждой эпохи для улучшения обучения

## Шаг 5: Загрузка датасета
Функция загружает датасет в формате YOLO (Roboflow). Для каждого изображения:

Считывается файл .txt с аннотацией

Извлекается ID класса объекта

Изображение изменяется до размера 32×32

Сохраняется в массив вместе с меткой

## Шаг 6: Загрузка и предобработка данных
Датасет загружается с Kaggle

Метки преобразуются из текстового формата в числовой

Значения пикселей нормализуются в диапазон [0, 1]

Данные разделяются на обучающую (80%) и тестовую (20%) выборки

Метки преобразуются в one-hot векторы для обучения

## Шаг 7: Построение архитектуры модели

Создается нейронная сеть со следующей архитектурой:

Encoder (3 блока): каждый блок содержит 2 сверточных слоя + Batch Normalization + MaxPooling

Bottleneck: сверточный блок с 256 фильтрами

Global Average Pooling: усреднение по пространственным измерениям

Классификационная голова: 2 полносвязных слоя с Dropout для регуляризации

## Шаг 8: Компиляция модели
Модель компилируется с:

Оптимизатором Adam: адаптивный метод оптимизации

Learning rate = 0.001: скорость обучения

Функцией потерь categorical_crossentropy: для многоклассовой классификации

Метрикой accuracy: точность классификации

## Шаг 9: Настройка callbacks

Настраиваются колбэки для автоматического управления обучением:

EarlyStopping: останавливает обучение при отсутствии улучшений на валидации

ReduceLROnPlateau: уменьшает скорость обучения при застое в обучении

## Шаг 10: Обучение модели

Модель обучается в течение 50 эпох:

На каждой эпохе данные проходят через генератор с аугментацией

После каждой эпохи оценивается точность на валидационной выборке

Колбэки автоматически регулируют процесс обучения

## Шаг 11: Визуализация процесса обучения
Строятся графики изменения функции потерь и точности в процессе обучения, что позволяет оценить:

Сходимость модели

Наличие переобучения (расхождение train и val кривых)

Качество обучения

## Шаг 12: Оценка качества модели

Вычисляется общая точность на тестовой выборке
