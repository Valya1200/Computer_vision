# Лабораторная работа №4 

Цель: Разработать классификатор

## ШАГ 1 Загрузка данных 
За основу был выбран датасет "Обнаружение дорожных знаков" на kaggle. Датасет состоит из 15 классов: Зеленый свет, красный свет, ограничение скорости 10, ограничение скорости 100, ограничение скорости 110, ограничение скорости 120, ограничение скорости 20, ограничение скорости 30, ограничение скорости 40, ограничение скорости 50, ограничение скорости 60, ограничение скорости 70, ограничение скорости 80, ограничение скорости 90, стоп

Процесс загрузки: автоматическая загрузка через библиотеку kagglehub или ручное указание пути,чтение файла data.yaml для получения названий классов, обход папок train, test, valid для сбора изображений и соответствующих label-файлов, каждое изображение загружается через OpenCV, конвертируется из BGR в RGB и изменяется до размера 224×224 пикселя, из label-файлов извлекаются class_id, которые сопоставляются с названиями классов
Результат загрузки: Массив изображений и массив текстовых меток классов

## Шаг 2. Предобработка данных
Нормализация: Изображения преобразуются из целочисленного диапазона [0, 255] в float32 [0, 1]
Кодирование меток: Текстовые названия классов преобразуются в числовые индексы с помощью LabelEncoder
Разделение выборки:Train: 70% всех данных (обучение модели), Validation: 15% (настройка гиперпараметров и early stopping), Test: 15% (финальная оценка качества)

Стратификация: Используется параметр stratify=y для сохранения пропорций классов во всех выборках (важно при дисбалансе классов)
One-hot encoding: Целевые метки преобразуются в категориальный формат для функции потерь categorical_crossentropy

## Шаг 3: Определение функций аугментации

Назначение: Искусственное увеличение разнообразия обучающей выборки для борьбы с переобучением

Используемые трансформации (для обучения):
HorizontalFlip	50%	Отражение по горизонтали
VerticalFlip	25%	Отражение по вертикали
ShiftScaleRotate	70%	Сдвиг (до 20%), масштабирование (до 20%), поворот (до 30°)
RandomBrightnessContrast	80% (в 90% случаев)	Изменение яркости и контраста
HueSaturationValue	80% (в 90% случаев)	Изменение цветового тона, насыщенности, яркости
GaussNoise	50%	Добавление случайного шума
Blur	30%	Размытие изображения
CoarseDropout	30%	Вырезание случайных прямоугольных областей

## Шаг 4. Веса классов
Проблема: Дисбаланс классов 
Решение: Автоматический расчет весов классов на основе частоты встречаемости:
Количество примеров	Вес класса
< 10	7.0
< 20	5.0
< 50	3.0
< 100	2.0
< 200	1.5
≥ 200	1.0
Применение: Веса передаются в модель при обучении, заставляя её "уделять больше внимания" редким классам

## Шаг 6. Двухфазное обучение
### Фаза 1: Обучение классификатора
Заморожена: Вся базовая модель EfficientNetB0
Обучается: Только добавленные Dense слои
Скорость обучения: 0.001
Эпохи: до 40 (с early stopping)
Цель: Быстро обучить классификатор на извлеченных признаках

### Фаза 2: Тонкая настройка (Fine-tuning)
Разморожены: Последние слои базовой модели (с 100-го слоя)
Заморожены: Ранние слои (0-99), отвечающие за базовые признаки
Скорость обучения: 0.0005 (меньше для аккуратной настройки)
Эпохи: до 60
Цель: Адаптировать признаки высокого уровня под специфику датасета
Callbacks:
EarlyStopping: Остановка при отсутствии улучшения валидационной точности (patience=15)
ReduceLROnPlateau: Уменьшение скорости обучения при застое (patience=5, factor=0.5)
ModelCheckpoint: Сохранение лучшей модели по валидационной точности
