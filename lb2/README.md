# Лабораторная работа №2 Визуальная одометрия

Цель: Разработать систему визуальной одометрии (навигации) по группе фотографий.

# ШАГ 1: Детектирование ключевых точек и вычисление SIFT дескрипторов
## 1.1. Предобработка данных
Загружено 6 последовательных изображений размером 320×240 пикселей. Конвертация в градации серого для ускорения обработки

## 1.2. Детектирование ключевых точек (Harris Corner Detector)

![window](https://github.com/Valya1200/Computer_vision/blob/main/images/window.png?raw=true)

Алгоритм:
1. Вычисление градиентов Ix, Iy с помощью свертки Собеля 

Оператор Собеля — это дискретный дифференциальный оператор, вычисляющий приближение градиента яркости изображения. Оператор вычисляет градиент яркости изображения в каждой точке. Так находится направление наибольшего увеличения яркости и величина её изменения в этом направлении.

![Sobel](https://github.com/Valya1200/Computer_vision/blob/main/images/mask_Sobel.png?raw=true)

(Gy,Gx) Маски Собеля

![Sobel](https://github.com/Valya1200/Computer_vision/blob/main/images/gradient_Sobel.png?raw=true)

2. Построение матрицы отклика углов: M = [[Ixx, Ixy], [Ixy, Iyy]]
3. Вычисление минимального собственного значения матрицы M
Анализ собственных значений λ₁ и λ₂ матрицы M:
Если λ₁ и λ₂ малы → плоская область
Если λ₁ >> λ₂ или λ₂ >> λ₁ → край
Если λ₁ и λ₂ велики → угол
4. Поиск локальных максимумов отклика
5. Выборка топ-N углов (N=30)
Результат: Найдено 180+ ключевых точек по всем изображениям

![detect_all](https://github.com/Valya1200/Computer_vision/blob/main/images/detect_all.png?raw=true)


# ШАГ 2: Сопоставление точек

## 2.1 Вычисление дескрипторов (SIFT-подобные)

SIFT (Scale-Invariant Feature Transform) — это алгоритм для выделения и описания ключевых точек, инвариантный к масштабу и повороту.

Для каждой ключевой точки (16×16 патч):
1. Вычисление градиентов gx, gy в патче
2. Разбиение на 4×4 блоков (по 4×4 пикселя)
3. Построение 8-биновой гистограммы ориентаций в каждом блоке

![desk](https://github.com/Valya1200/Computer_vision/blob/main/images/desck_sift.png?raw=true)

4. Нормализация L2 + клиппинг (0.2) + перенормировка
Нормализация L2 необходимо для определения всех значений в диапазон 0-1, и их сумма квадратов = 1.
clipping (ограничение) необходим, чтобы один очень сильный градиент не доминировал над всеми остальными. Это делает дескриптор устойчивее к изменениям освещения.
5. Итоговый дескриптор: 128-мерный вектор
Результат: 100+ валидных дескрипторов на изображение


#  ШАГ 3: Построение траектории
## 3.1 Сопоставление признаков (Lowe's ratio test)
Критерий: dist1 < 0.75 × dist2 (где dist1, dist2 — ближайшие соседи)
Результат: 20-50 совпадений между соседними кадрами
